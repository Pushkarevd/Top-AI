# КИМ-4.1. Тестирование по теме «Инфраструктура больших данных»

## Назначение оценочного средства

Оценочное средство предназначено для контроля и оценки результатов освоения дисциплины «Большие данные» в части выбора технологий и компонентов инфраструктуры для работы с большими данными.

Тестирование проверяет способность обучающегося:
- выбирать компоненты единого технологического стека для работы с большими данными;
- обосновывать выбор технологий для конкретных сценариев использования;
- понимать принципы масштабируемости, надёжности и отказоустойчивости инфраструктуры;
- различать слои хранения данных и их назначение в архитектуре Data Lake / Data Warehouse.

## Проверяемые результаты обучения и индикаторы

| Компетенция | Индикатор | Уровень | Дескриптор освоения |
| :---------: | :-------: | ------------------- |
| **BD-5.** Способен применять технологии организации инфраструктуры больших данных | BD-5.1 | С | Осуществляет выбор направления вспомогательных технологических решений для формирования единого стека работы с большими данными для решения поставленной задачи |

## Условия и продолжительность выполнения

- **Форма контроля:** тестирование
- **Продолжительность:** 45 минут
- **Количество заданий:** 20 вопросов
- **Типы заданий:**
  - 15 вопросов с выбором одного правильного ответа (1 балл за каждый);
  - 5 вопросов с выбором нескольких правильных ответов (2 балла за каждый).
- **Максимальный балл:** 25
- **Формат проведения:** онлайн (LMS) или письменно
- **Разрешённые материалы:** не разрешается использование внешних источников, документации, генеративного ИИ

## Формулировка задания и исходные материалы

### Часть 1. Вопросы с выбором одного правильного ответа (15 вопросов)

**Вопрос 1.** Какой компонент инфраструктуры больших данных отвечает за буферизацию и передачу потоковых данных между источниками и потребителями?

a) Data Warehouse  
b) Message Broker (например, Apache Kafka)  
c) Data Lake  
d) BI-инструмент  

**Правильный ответ:** b

---

**Вопрос 2.** Что из перечисленного НЕ является характерным признаком Data Lake?

a) Хранение данных в сыром формате  
b) Схема применяется при чтении (schema-on-read)  
c) Данные хранятся только в реляционном формате  
d) Поддержка неструктурированных данных  

**Правильный ответ:** c

---

**Вопрос 3.** Какой слой Data Lake содержит очищенные и стандартизированные данные?

a) Bronze (Raw)  
b) Silver (Cleaned)  
c) Gold (Curated)  
d) Platinum (Production)  

**Правильный ответ:** b

---

**Вопрос 4.** Какая технология наиболее подходит для хранения и анализа временных рядов телеметрии IoT-устройств?

a) MongoDB  
b) InfluxDB / TimescaleDB  
c) Apache Hadoop HDFS  
d) Redis  

**Правильный ответ:** b

---

**Вопрос 5.** Что такое горизонтальное масштабирование (scaling out)?

c) Увеличение объёма дискового пространства  
d) Оптимизация запросов к базе данных  

**Правильный ответ:** b

---

**Вопрос 6.** Какой инструмент предназначен для оркестрации конвейеров данных (data pipelines)?

a) Apache Kafka

c) Apache Spark

**Правильный ответ:** b

**Вопрос 7.** Какая технология обеспечивает потоковую обработку данных в реальном времени?

---

a) Увеличение частоты процессора

**Вопрос 12.** Какая технология наиболее подходит для кэширования результатов агрегаций в реальном времени?

**Вопрос 13.** Что такое «витрина данных» (Data Mart)?

### Часть 2. Вопросы с выбором нескольких правильных ответов (5 вопросов)

**Вопрос 16.** Какие из перечисленных технологий относятся к Message Brokers? (выберите 2)

a) Apache Kafka  
b) Apache Spark  
c) RabbitMQ  
d) Apache Hive  
e) MinIO  

**Правильные ответы:** a, c

---

**Вопрос 17.** Какие характеристики относятся к слою Gold (Curated) в Data Lake? (выберите 3)

a) Данные очищены и проверены  
b) Хранятся бизнес-агрегаты  
c) Данные в сыром формате  
d) Оптимизировано для аналитических запросов  
e) Схема применяется при записи  

**Правильные ответы:** a, b, d

---

**Вопрос 18.** Какие механизмы обеспечивают надёжность инфраструктуры? (выберите 3)

a) Репликация данных  
b) Checkpointing в потоковой обработке  
c) Увеличение объёма оперативной памяти  
d) Retry policies при сбоях  
e) Использование только одного типа хранилища  

**Правильные ответы:** a, b, d

---

**Вопрос 19.** Какие сценарии использования подходят для Apache Kafka? (выберите 2)

a) Приём событий кликов в реальном времени  
b) Хранение исторических отчётов за 5 лет  
c) Буферизация телеметрии IoT-устройств  
d) Визуализация дашбордов  
e) Выполнение сложных SQL-запросов  

**Правильные ответы:** a, c

---

**Вопрос 20.** Какие компоненты входят в типовой стек для потоковой аналитики? (выберите 3)

a) Message Broker (Kafka)  
b) Stream Processor (Spark Streaming / Flink)  
c) Batch Processor (Spark Batch)  
d) Time-Series Database / Data Warehouse  
e) ETL-инструмент для пакетной загрузки  

**Правильные ответы:** a, b, d

## Требования к формату сдачи

- Тестирование проводится в электронной форме через LMS или на бумажном носителе.
- Обучающийся отмечает выбранные ответы в бланке тестирования.
- Для вопросов с несколькими правильными ответами засчитываются только случаи, когда выбраны **все** правильные ответы и не выбраны неправильные.

| Диапазон баллов | Оценка | Описание |
| :-------------: | ------ | -------- |

## Правила использования внешних источников, программных инструментов и генеративного ИИ

- При прохождении тестирования **запрещено** использование:
  - внешней документации и поисковых систем;
  - генеративного ИИ и автоматизированных средств поиска ответов;
  - помощи других лиц.
- Разрешается использовать только черновик для заметок.
- Нарушение правил академической честности влечёт аннулирование результата.

## Требования к воспроизводимости и самостоятельности выполнения работы

- Тестирование выполняется индивидуально.
- Варианты вопросов могут быть перемешаны для разных обучающихся.
- При выявлении совпадения паттернов ответов без возможности объяснить выбор технологии результат аннулируется.
