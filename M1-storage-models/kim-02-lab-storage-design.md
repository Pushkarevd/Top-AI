# КИМ-1.2. Лабораторная работа «Проектирование модели хранения данных»

Соответствует занятию 1.3 учебного плана дисциплины.

## Назначение оценочного средства

Оценочное средство проверяет способность обучающегося работать с реляционными СУБД в режимах OLTP и OLAP, писать аналитические SQL-запросы. Аналиизровать планы выполнения. Выбирать способ хранения и сущности для анализа.

## Проверяемые результаты обучения и индикаторы

| Индикатор | Уровень | Дескриптор освоения                                                                                                                                                                            |
|-----------| :-----: |------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| BD-3      | С | Разрабатывает, отлаживает и тестирует прикладные решения с элементами ИИ с примением различных технологий хранения структурированных данных, оценивает качество построенных прикладных решений |
| PL-1      | С | Способен применять язык программирования Python для решения задач в области ИИ

## Условия и продолжительность выполнения

* Форма занятия: лабораторное занятие, 4 ак. ч.
* Скрипты для заполнения БД: предоставляются в рамках методических указаний
* Сервисы хранения (СУБД), обработки и вспомогательные компоненты рекомендуется развернуть в Docker-контейнерах с использованием конфигурации `docker-compose`.


## Формулировка задания и исходные материалы

Полный текст задания: [Занятие 1.2. Проектирование модели хранения данных](attachments/lab_1_1.md).

Задачи:
- Изучить теоретические основы OLTP и OLAP систем
- Спроектировать и реализовать схему данных для OLTP-системы
- Спроектировать и реализовать схему данных для OLAP-системы (хранилище данных)
- Провести сравнительный анализ производительности запросов
- Научиться читать и анализировать планы выполнения запросов
- Оптимизировать запросы на основе анализа планов выполнения

## Требования к формату сдачи

Отчёт по лабораторной работе должен содержать:

1. **Титульный лист** (номер работы, название, ФИО, группа)

2. **Цель работы**

3. **Схемы данных:**
    - ER-диаграмма OLTP-системы
    - Схема «звезда» OLAP-системы

4. **Результаты выполнения:**
    - Скриншоты выполненных запросов
    - Планы выполнения запросов с анализом
    - Таблица сравнения производительности

5. **Анализ планов выполнения:**
    - Ответы на вопросы из Части 4
    - Предложения по оптимизации

6. **Выводы:**
    - Когда целесообразно использовать OLTP
    - Когда целесообразно использовать OLAP
    - Какие преимущества показала каждая архитектура

## Правила использования внешних источников, программных инструментов и генеративного ИИ

Разрешено использование документации, открытых библиотек и генеративного ИИ при написании кода и отчета. Обучающийся должен уметь объяснить работу любого фрагмента представленного кода на защите.

## Требования к воспроизводимости и самостоятельности выполнения работы

Решение должно разворачиваться и запускаться по приложенной инструкции на чистом окружении. Идемпотентность процесса подтверждается повторным запуском за ранее обработанный интервал.

Работа выполняется индивидуально. Совпадение исходного кода и отчета с работами других обучающихся является основанием для возврата работы на повторное выполнение с другим источником данных.
