# Контрольно-измерительный материал
## Тест к занятию 3.2 «Распределённая обработка данных; MPP-системы; Экосистема Apache Hadoop; Spark/Trino»

---

## 1. Назначение оценочного средства

Комплект контрольно-измерительных материалов предназначен для текущего контроля уровня сформированности компетенции **BD-4.1** у обучающихся по дисциплине «Технологии хранения и обработки больших данных». Оценочное средство охватывает теоретические основы распределённой обработки больших данных: архитектуры MPP-систем, компоненты экосистемы Apache Hadoop (HDFS, YARN), модель вычислений Apache Spark и роль Apache Trino в архитектуре Data Lakehouse.

КИМ позволяет проверить способность обучающегося:
- понимать принципы организации распределённого хранения и параллельной обработки больших данных;
- различать роли ключевых компонентов экосистемы Apache Hadoop (HDFS, YARN);
- применять основные концепции Apache Spark (RDD, DataFrame, ленивые вычисления, DAG, трансформации и действия);
- понимать назначение Apache Trino в связке со Spark и Apache Iceberg;
- выбирать адекватные технологии обработки больших данных под заданные требования прикладной системы ИИ.

## 3. Условия выполнения

- **Форма проведения:** текущий контроль в форме тестирования на лекционно-практическом занятии.
- **Продолжительность:** 40–60 минут.
- **Формат:** 26 вопросов, из которых 15 — с одиночным выбором ответа, 11 — с множественным выбором ответа.
- **Технические требования:** персональный компьютер обучающегося с доступом к системе дистанционного тестирования (или бумажный бланк).
- **Допускается:** использование официальной документации Apache Spark, Hadoop, Iceberg, Trino (по согласованию с преподавателем).

## 5. Критерии и рубрики оценивания

### 5.1. Начисление баллов

| Тип вопроса | Балл за правильный ответ | Балл за неправильный ответ |
|-------------|--------------------------|----------------------------|
| Одиночный выбор (15 вопросов) | 1 | 0 |
| Множественный выбор (11 вопросов) | 1,5 | 0 |

*Примечание:* для вопросов с множественным выбором ответ считается правильным только при полном совпадении выбранных вариантов с ключом (частичное совпадение не оценивается).

### 5.2. Шкала оценивания

| Баллы | Оценка |
|-------|--------|
| 30–31,5 | Отлично |
| 24–29 | Хорошо |
| 17–23 | Удовлетворительно |
| Менее 17 | Неудовлетворительно |

### 5.3. Рубрики оценивания по дескриптору компетенции BD-4.1

| Уровень освоения | Критерии |
|------------------|----------|
| **Не освоен** (менее 17 баллов) | Обучающийся не различает ключевые компоненты экосистемы Hadoop, не понимает принципов распределённого хранения и параллельной обработки, не может обосновать выбор технологий |
| **Освоен на базовом уровне** (17–23 балла) | Обучающийся знает основные компоненты HDFS, YARN, Spark, но испытывает затруднения при обосновании выбора технологий и понимании их взаимодействия |
| **Освоен на продвинутом уровне** (24–29 баллов) | Обучающийся уверенно ориентируется в экосистеме, понимает роль Spark и Trino в архитектуре Lakehouse, способен обосновать выбор технологий |
| **Освоен на высоком уровне** (30–33 балла) | Обучающийся демонстрирует глубокое понимание всех компонентов, способен связать теорию с практикой лабораторных работ, обосновать архитектурные решения |

## 6. Правила использования внешних ресурсов и генеративного ИИ

**Разрешено:**
- Использование официальной документации Apache Hadoop, Spark, Iceberg, Trino.
- Консультации с генеративным ИИ (ChatGPT, YandexGPT, GigaChat) для уточнения определений и объяснения концепций **в рамках самостоятельной подготовки к тесту**.

**Запрещено:**
- Использование внешних ресурсов и ИИ **в процессе выполнения теста** (тест проводится в режиме закрытого доступа).,

**Требования:**
- Тест выполняется индивидуально.
- Обучающийся должен быть готов устно пояснить ответы на вопросы теста при защите лабораторных работ (занятия 3.3 и 3.4).

---

## 7. Формат сдачи

- Тест выполняется в электронной форме через систему дистанционного тестирования университета (или на бумажном бланке).
- Результаты фиксируются в журнале успеваемости.
- При возникновении спорных ситуаций обучающийся может апеллировать к преподавателю с обоснованием своего ответа.
