# КИМ-3.1. Контрольная работа по теме «Технологии и инструменты обработки данных»

[Контрольная работа](attachments/appendix-01-control-work_M3.pdf).

## Назначение оценочного средства
Контрольная работа по модулю 3 «Технологии и инструменты обработки данных» предназначена для рубежного контроля студентов и оценки уровня сформированности профессиональных компетенций в области распределенной обработки и управления качеством больших данных.

Контрольная работа является составной частью учебного процесса по дисциплине и позволяет:

Оценить теоретические знания студентов о современных технологиях обработки больших данных (Apache Spark, Apache Airflow, Apache Kafka, форматы хранения, системы управления метаданными).

Проверить практические навыки разработки ETL-конвейеров, написания Spark-приложений, создания DAG в Airflow и реализации проверок качества данных.

Оценить способность студентов к анализу производительности, выявлению узких мест и оптимизации решений.

Выявить навыки работы с генеративным искусственным интеллектом для решения профессиональных задач (генерация данных, проверка кода, оптимизация решений).

## Формулировка задания и исходные материалы
Контрольная работа состоит из трех блоков, каждый из которых проверяет определенный аспект профессиональной подготовки:

Блок 1. Тестовая часть (14 баллов)
Назначение: Проверка теоретических знаний о технологиях обработки больших данных.

Содержание: 14 вопросов с выбором ответа, охватывающих:

  Основы распределенной обработки (Spark, Hadoop, MapReduce).

  Пакетная и потоковая обработка (Kafka, Spark Streaming).

  Форматы хранения (Parquet, ORC).

  Управление метаданными и качеством данных.

  Инструменты оркестрации (Airflow).

Формат: Выбор одного правильного ответа из четырех предложенных.

Время выполнения: 15-20 минут.

Блок 2. Практическое задание (16 баллов)
Назначение: Проверка практических навыков разработки ETL-конвейеров и управления качеством данных.

Содержание: Три задачи, требующие:

  Проектирования архитектуры конвейера обработки данных.

  Написания DAG для Apache Airflow.

  Разработки Spark-приложения.

  Анализа производительности и оптимизации.

  Реализации проверок качества данных.

Формат: Развернутые письменные ответы с кодом на Python/PySpark.

Время выполнения: 45-60 минут.

Блок 3. Задания с использованием искусственного интеллекта (5 баллов)
Назначение: Проверка навыков использования генеративного ИИ в профессиональной деятельности инженера данных.

Содержание: Две задачи:

  Генерация тестовых данных с помощью ИИ.

  Проверка кода с помощью ИИ и его оптимизация.

Формат: Формулировка запросов к ИИ, получение ответов, анализ и внесение исправлений.

Время выполнения: 15-20 минут.
