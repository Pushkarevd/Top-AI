# Проектная работа

Командная проектная работа — интегральный КИМ, позволяющий проверить несколько компетенций и индикаторов в условиях комплексной практической задачи.

В составе команды обучающиеся проектируют и реализуют прототип системы сбора, хранения и обработки данных, распределяют задачи между участниками и ИИ-инструментами, фиксируют критерии качества и представляют результаты.

**Форма контроля:** рубежный контроль, защита проектной работы.

**Проверяемые индикаторы:** BD-2.2, BD-3.1, BD-4.1, BD-4.3, BD-5.1, PL-1.3, SS-2.1 (уровни Б–П).

## Назначение оценочного средства
Контрольно-измерительные материалы (КИМ) предназначены для оценки уровня сформированности компетенций студентов в ходе выполнения проектной работы по дисциплине «Технологии хранения и обработки больших данных».
Цель проектной работы: освоить на практике полный цикл работы инженера данных: от проектирования архитектуры хранения и сбора данных из различных источников до построения прогностической модели и оценки качества всего решения, с использованием современных технологий больших данных, машинного обучения и искусственного интеллекта.
## Проверяемые результаты обучения и индикаторы

| BD-3.1 | С | Проектирует архитектуру хранения. Реализует схему больших данных и загрузку данных |
| BD-4.1 | П | Обосновывает выбор технологий обработки больших данных. Реализует распределенную обработку и организует централизованное хранилище данных |
| BD-4.3 | П | Проектирует и отлаживает конвейеры обработки данных, настраивает оркестрацию, мониторинг, проверки качества и Data Lineage, оценивает надежность и производительность решения |
| BD-5.1 | С | Студент формирует и обосновывает полный технологический стек проекта, проектирует инфраструктуру с учетом масштабируемости |

## Условия и продолжительность выполнения
Тип проекта: командный (3-4 человека). Групповой проект с распределением ролей.
Общая трудоемкость: 18 ак.ч.
Формат сдачи: Презентация (10-15 минут) с демонстрацией работающего прототипа. Отчет.

## Формулировка задания и исходные материалы
Спроектировать и реализовать прототип инфраструктуры для сбора, хранения и обработки больших данных в рамках выбранной предметной области. Сформировать ML-модель на базе собранных данных.
Задачи:
1.	Выбрать предметную область и сформулировать бизнес-задачу, используя генеративный ИИ (ChatGPT, YandexGPT, GigaChat).
2.	Спроектировать архитектуру хранения и обработки данных.
3.	Реализовать ETL/ELT-конвейер для интеграции данных из разнородных источников.
4.	Проверить код ETL-процессов с помощью больших языковых моделей.
5.	Организовать хранение данных с использованием различных моделей (реляционная, NoSQL, Data Lake).
6.	Реализовать распределенную обработку данных с использованием Apache Spark.
7.	Внедрить механизмы контроля качества данных (Data Quality) на всех этапах процесса.
8.	Настроить оркестрацию конвейера с использованием Apache Airflow.
9.	Обеспечить прослеживаемость данных (Data Lineage) с использованием OpenMetadata.
10.	Сформировать модель на основе подготовленных данных и применить ее для решения задачи. 
11.	Подготовить проектную документацию и защитить решение перед комиссией.
